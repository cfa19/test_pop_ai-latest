# Enrich & Release Results

When a beneficiary completes an activity, the results enter a review process. Consultants
enrich results with personalized feedback before releasing them.

---

## Result detail page

Open a result from the Consultant Results Dashboard. The detail
page includes:

### Header

- **Beneficiary** profile (name, email, role)
- **Activity** name and type
- **Completion date** and **time spent**
- **Score** badge
- **PDF export** button
- **Release controls**

### Intervention alerts

Automatically generated alerts highlight areas that need attention:

| Severity | Color | Meaning |
|---|---|---|
| **Critical** | Red | Immediate action required |
| **High** | Orange | Significant concerns |
| **Medium** | Yellow | Minor issues to review |
| **Low** | Blue | Informational notes |

---

## Tabbed content

The detail page has five tabs:

### 1. Summary

A rendered view of the activity results. Shows the key outputs in a structured, readable
format.

### 2. Details

Raw data in JSON format for technical analysis. Collapsible sections for easy navigation.

### 3. Files

File attachments uploaded during the activity. View, download, or manage files.

### 4. Enrichment

This is where the consultant adds value. Enrichment sections are divided into **private** and
**shared** content:

**Private (consultant only)**

| Section | Description |
|---|---|
| **Confidential notes** | Internal notes that are **never** shared with the beneficiary. Use them for follow-up reminders, risk observations, or coordination notes with other consultants. |

**Shared (visible to the beneficiary after release)**

| Section | Description |
|---|---|
| **Recommendations** | Structured recommendations the beneficiary will see in their results page |
| **Beneficiary narrative** | A motivational story summarizing the beneficiary's achievements |
| **Next steps** | Actionable steps with difficulty badges and estimated times |
| **Gamification** | Badges, levels, and achievements to motivate the beneficiary |

Everything in the shared sections will be visible to the beneficiary once results are released. Only **Confidential notes** remain private. Double-check the content before releasing.

All enrichment data is auto-saved. A save indicator confirms the current status.

### 5. Beneficiary preview

A preview of exactly what the beneficiary will see when the results are released, including:

- Hero section with trophy icon
- Progress overview with progress bar
- Motivational narrative
- Next steps with difficulty badges
- Consultant recommendations
- Gamification panel (badges, level, achievements)

Use this tab to verify the beneficiary experience before releasing.

---

## Release controls

When you are ready to publish the results to the beneficiary:

| Action | Description |
|---|---|
| **Release** | Immediately publish results to the beneficiary (with optional message) |
| **Schedule** | Set a specific date and time for automatic release |

A status badge shows the current release state: *Pending review*, *Released*, or *Scheduled*.
