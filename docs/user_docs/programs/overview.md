# Programs Overview

A program is a structured sequence of activities grouped into a learning path. Programs allow
consultants and organizations to guide beneficiaries through a curated set of exercises in a
defined order.

---

## How programs work

1. A consultant or admin **creates a program** and adds activities to it.
2. The program is **assigned** to one or more beneficiaries.
3. Beneficiaries work through the activities **sequentially**.
4. Progress is tracked per activity and for the program as a whole.
5. When all activities are completed, the consultant can **validate** the program.

---

## Program properties

| Property | Description |
|---|---|
| **Title** | Name of the program |
| **Description** | What the program covers and its objectives |
| **Category** | Classification for filtering |
| **Activities** | Ordered list of activities included |
| **Credit cost** | Total credits required (sum of all activity costs) |
| **Price** | Optional program price (may differ from total credit cost) |
| **Public** | Whether the program is visible for self-enrollment |

---

## Who can access programs

| Role | Capabilities |
|---|---|
| **Org Admin** | Create, edit, assign programs within their organization |
| **Consultant** | Create, edit, assign programs to their beneficiaries |
| **Beneficiary** | View and complete assigned programs |
| **Individual User** | Browse and enroll in public programs (via the activity catalog) |
